# Техническое задание: сайт с промокодами магазинов

## 1. Цель проекта

Сайт хранит и публикует промокоды магазинов. Пользователь может найти активный промокод, отфильтровать его по магазину, группе, скидке и минимальной сумме заказа. Администратор управляет магазинами, группами и промокодами, а API отдаёт данные для внешних клиентов.

## 2. Роли пользователей

### Администратор

1. Создаёт, редактирует и удаляет магазины, группы и промокоды.
2. Видит полный промокод, историю изменений и аналитику кликов.
3. Может деактивировать промокоды вручную через API/action и админку.
4. Может импортировать/экспортировать промокоды через django-import-export.

### Покупатель / авторизованный пользователь

1. Просматривает список активных и неистёкших промокодов.
2. Может раскрыть промокод через HTML-страницу или API `reveal`.
3. При раскрытии промокода система записывает клик для аналитики.
4. Может искать и фильтровать промокоды по магазину, группе, скидке, минимальной сумме заказа.

### Гость

1. Просматривает только активные и неистёкшие промокоды.
2. Видит скрытый код `•••••`, но не полный промокод.
3. Может пользоваться поиском и фильтрами API без раскрытия кода.

## 3. Бизнес-логика и валидация

1. Активный промокод не может иметь дату окончания в прошлом.
2. Скидка должна быть в диапазоне от 1 до 100%.
3. Код промокода нормализуется в верхний регистр.
4. Код должен быть не короче 5 символов и состоять из букв, цифр, дефиса или подчёркивания.
5. Для одного магазина не может существовать два активных промокода с одинаковым кодом.
6. Количество использований не может быть больше лимита применений.
7. Сайт магазина должен начинаться с `http://` или `https://`.

## 4. Оптимизация запросов

Использованы `select_related('shop', 'created_by')` и `prefetch_related('groups')` в HTML-views и API. В API и админке добавлены аннотации `Count('clicks')` и `Count('promos')`.

## 5. API и сериализаторы

`PromoSerializer` использует:

- `SerializerMethodField`: `shop_name`, `group_names`, `status`, `discount_label`, `public_code`;
- контекст сериализатора: `reveal_codes`, `role`, `request` для скрытия/открытия промокода;
- model-level validation через `Promo.clean()`.

## 6. Фильтрация Django Filter

Фильтры API `/api/promos/`:

- `shop` — id магазина;
- `manufacturer` / `shop_name` — поиск по названию магазина;
- `category` / `group` — id группы промокодов;
- `discount_min`, `discount_max` — диапазон скидки;
- `price_min`, `price_max` — диапазон минимальной суммы заказа;
- `expires_from`, `expires_to` — диапазон даты окончания;
- `is_active` — активность.

## 7. Sentry

Sentry подключается через переменную окружения `SENTRY_DSN`. Если переменная пустая, Sentry не активируется.

## 8. Django Silk

В режиме `DEBUG=True` доступен профайлер `/silk/` для анализа SQL-запросов и времени выполнения.

## 9. Celery и Mailhog

Celery использует Redis. Задача `deactivate_expired_promos_task` каждый час деактивирует истёкшие промокоды. Mailhog используется для локальной проверки писем на `http://localhost:8025`.

## 10. OAuth2

Добавлена интеграция `social-auth-app-django` для входа через Google OAuth2. Для работы нужно указать `GOOGLE_OAUTH2_KEY` и `GOOGLE_OAUTH2_SECRET`.

## 11. Postman

Файл `postman_collection.json` содержит коллекцию запросов для проверки API.
